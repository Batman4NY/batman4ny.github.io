Cisco Secure AI Factory, implemented in a home lab.

A working rendering of the Cisco Secure AI Factory with NVIDIA reference architecture on OCI Free Tier + NVIDIA NIM Free Tier — $0/mo, perpetual, reproducible. Real code, live URLs, published guides.

§ Why this exists The question a running lab actually answers

"What would Cisco Secure AI Factory look like at our scale?"

Every Cisco DC SE gets some version of that question every week — from a customer, from a peer, from someone at a conference. A slide deck doesn't answer it. A slide deck describes what a customer could build. A running lab shows what is running now, in the same product line-up Cisco and NVIDIA co-market to enterprise DC customers.

This site is that running lab. Every artifact here has a live URL, a public repository, or a published guide. The AI Defense demo is reachable now. The SAIF stack shows implementation status honestly, layer by layer — partial, adjacent, or planned — no overclaiming. The roadmap lays out T1–T4 rollout tiers side-by-side with the enterprise version.

§ Why the free tier matters The pattern reproduces at $0/mo — the enterprise version doesn't have to

The full enterprise SAIF is serious infrastructure — Cisco UCS + NVIDIA RTX PRO 4500 Blackwell GPUs, Nexus 9000 fabric with Silicon One G300/P200, Cisco Hybrid Mesh Firewall on NVIDIA BlueField DPUs. Not something an SE spins up over a weekend.

But the pattern is reproducible on $0/mo substrates: OCI Free Tier compute (2× AMD Micro + 1× ARM Ampere with 4 OCPU / 24 GB), NVIDIA NIM Free Tier hosted inference (5K credits/mo, replenishing), Cloudflare DNS, Caddy for auto-TLS via Let's Encrypt, GitHub Pages for static sites. Every layer that can be implemented on free-tier substrates is implemented. Where the reference requires enterprise hardware or paid services, the layer is honestly labeled — the reference architecture page spells it out.

§ Why reproducibility is the point Any SE on the team can rebuild this — that's the whole thesis

Every artifact on this site is open source under Apache-2.0. The infrastructure setup is a chapter-by-chapter guide — thirteen chapters, ~10K words, from Oracle account signup through the last Caddy hostname. The AI Defense demo runs from docker-compose up. The ConsolePi build is a shopping list and 30 minutes of work.

No secret sauce. No "call me for the config." Any SE on the team who wants to reproduce this can — start-to-finish, over a weekend, at zero incremental cost. That's the whole point. When a peer asks how to build one, the answer is a URL, not a screen-share.

§ Why this and not Cloud Control The lab is the bench. Cloud Control will be the cockpit.

Cisco Cloud Control is Cisco's announced (pre-GA) AI-native platform for cross-domain security operations. When it ships, it's where customers will run every product on this site at production scale. This lab isn't a substitute for Cloud Control — it's the hands-on foundation SEs need to be fluent in the portfolio Cloud Control will orchestrate.

Build the stack here now. Be ready to operate it there when the cockpit ships. Full positioning on the roadmap page.

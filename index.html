§ Agent security Emerging cross-cutting pattern cuts across L2 + Security wrap · GTC 2026
Agent runtime + policy — where an autonomous agent runs and how it's governed status: implemented (LLM I/O guard) · planned (guardrails · agent runtime)

As LLM systems become agentic — invoking tools, calling other agents, taking actions with real consequences — a distinct security surface emerges that doesn't cleanly belong to any single SAIF layer. It runs at L2 (AI Runtime) for the runtime components and at the Security wrap for guardrails and policy. The GTC 2026 Cisco+NVIDIA co-announcement calls this out as a first-class integration pattern.

NVIDIA OpenShell
Agent development platform

Runtime + tooling for building autonomous agents. The substrate the agent lives on.

Sits at · L2 AI Runtime
NVIDIA Agent Toolkit
Agent runtime tools

Libraries + connectors for building agents that invoke tools and other agents. The plumbing.

Sits at · L2 AI Runtime
NVIDIA NeMo Guardrails
Runtime guardrails

Rails on what the LLM/agent can say + do. Programmable policy at inference time. Complements AI Defense.

Sits at · Security wrap
Cisco AI Defense
LLM I/O guard

Three-point inspection of input · tool args · output. Catches prompt injection, destructive tool calls, PII/credential exfil.

§ Adjacent tooling Practitioner substrate not in SAIF — but every lab needs it
ConsolePi OOB · Serial console

Out-of-band (OOB) serial console server on Raspberry Pi hardware. Terminates serial from Cisco Secure Firewalls (1200 / 3100 / 4200) via USB-C native CDC-ACM, and from legacy DC gear via RJ45 with USB-serial adapters. Used for initial device bring-up and management-plane recovery. Not part of SAIF — standard DC operator tooling.
